## Design Context

### Users
Developers using the dashboard to monitor their own AI-assisted work across projects. They are
checking personal usage patterns, not reporting to managers or clients, so the interface should
feel direct, lightweight, and personally useful.

### Brand Personality
Fun, personal, and non-corporate. The experience should feel like a tool made by and for working
developers rather than a polished enterprise analytics suite.

### Aesthetic Direction
Light mode with warmth and personality. Avoid corporate dashboard tropes, sterile enterprise UI,
and anything that reads like generic SaaS analytics. Favor expressive but still practical
presentation over compliance-first business styling.

### Design Principles
1. Keep the ledger useful first: the primary workflow is scanning project activity quickly.
2. Make analytics feel personal, not managerial or performative.
3. Prefer expressive details that add charm over enterprise polish that adds distance.
4. Keep controls and states obvious so the interface stays fast for repeat use.
5. Avoid corporate dashboard clichés even when presenting dense data.
